# 非機能要件（対象外項目）

- 認証・認可: API レベルの認証（API Key / Azure AD）は不要。今回の範囲では実装しない。
- レート制限: IP や API 単位のレート制御は対象外。
- 永続ストレージ: データベースや永続化メカニズムは不要。
- UI/フロントエンド: 専用のフロントエンドは作成しない。ブラウザから直接 JSON を確認する想定。
- 高可用性設計: Consumption Plan の標準動作以内で運用し、追加の HA 設計は行わない。
- 高負荷対応: 大量同時リクエストや高スループットの性能チューニングは対象外。
- 小数点対応: `A`,`B` の小数対応は行わない（整数のみ対応）。
- 複雑なエラーフォーマット: シンプルな HTTP ステータス + 簡易 JSON メッセージ以外は実装しない。
- IaC（ARM/Bicep/Terraform）: インフラ定義の自動化は今回の必須要件ではない。
- 外部監査・セキュリティスキャン: セキュリティ監査や自動スキャンは含めない。

※ 将来的に上記を必要とする場合は別途要件として追加する。
